# Contributing

Thanks for considering a contribution. This project benefits from engineers who care about building agent systems that work in production, not just in demos.

## What helps most

- **Errata and corrections.** Found something wrong, misleading, or outdated? Open an issue.
- **Real-world examples.** If you have a case where workflow beat agent design (or vice versa), share it in Discussions.
- **Code improvements.** Bug fixes, test coverage, clearer examples. Keep changes focused.
- **Diagram improvements.** Architecture diagrams that make concepts clearer.

## How to contribute

1. Fork the repo and create a branch from `master`.
2. Make your changes. Keep commits focused.
3. Run `make test` to verify tests pass.
4. Open a pull request with a clear description of what changed and why.

## Style

- Write for experienced engineers. No hand-holding, no hype.
- Code examples should be minimal and runnable.
- Prefer concrete failure modes over abstract advice.
- No framework worship. Concepts should survive tool churn.

## What we probably won't merge

- Framework-specific tutorials or vendor promotions
- Content that reads like marketing copy
- Changes that add complexity without clear engineering value

## Questions?

Open a Discussion or an issue. Both are welcome.
