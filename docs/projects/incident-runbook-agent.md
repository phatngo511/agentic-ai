# Incident Runbook Agent

*Coming soon -- this project is being built as part of Phase 2.*

An operational agent that inspects system signals, searches runbooks for matching procedures, proposes remediation steps, and requests human approval before executing any action.

**Teaches:** Human-in-the-loop architecture, approval gates, escalation policies, audit logging, bounded operational autonomy.

See [Chapter 5: Human-in-the-Loop as Architecture](../book/05-human-in-the-loop.md) for the concepts behind this project.
